Guía de despliegue · para el administrador

Poner oposicionesdeporte · Planes en producción

App interna que genera planes de preparación en .docx a partir del texto de un boletín. Se despliega en contenedores: la app más Caddy, que emite el certificado HTTPS automáticamente.

Ya está probado de extremo a extremo — construcción de la imagen, arranque del contenedor y login verificados. Solo queda que tú añadas la API key real (paso 3) y configures el dominio. No hace falta instalar Node ni LibreOffice en el sistema: la imagen los trae.

Antes de empezar

Necesitas un servidor Ubuntu/Debian con Docker y Docker Compose, un subdominio libre y los puertos 80 y 443 accesibles desde internet.

1

Apuntar el DNS

En tu proveedor de dominios, crea un registro A del subdominio hacia la IP pública del servidor y espera a que propague:

planes.tudominio.com   A   <IP_DEL_SERVIDOR>

$ dig +short planes.tudominio.com   # debe devolver la IP del servidor

Imprescindible antes de arrancar: Caddy pide el certificado a Let's Encrypt y necesita que el dominio ya resuelva al servidor.

2

Subir el proyecto

Por git o por copia de archivos. No copies ningún .env de otra máquina (por ejemplo el de pruebas en local): el servidor usa el suyo, que se crea en el paso siguiente.

# opción A — clonar el repositorio
$ git clone <URL_DEL_REPO> planes-app && cd planes-app

# opción B — descomprimir el zip enviado
$ unzip planes-app.zip && cd planes-app
3

Configurar el .env — aquí va la API key

$ cp .env.example .env
$ nano .env
VariableValor
ANTHROPIC_API_KEYLa clave real de la API de Anthropic (la que custodias tú).
SESSION_SECRETCadena larga aleatoria. Genérala con openssl rand -hex 32.
APP_USERSUsuarios del equipo: rosario:CLAVE,colab1:CLAVE2 (usuario:contraseña, separados por comas).
DOMINIOTu subdominio, p. ej. planes.tudominio.com.
NODE_ENVDéjalo en production (ya viene puesto). Activa las cookies seguras; imprescindible detrás de Caddy.

Guarda con Ctrl+O, Enter, Ctrl+X. Nunca subas el .env a un repositorio (ya está en .gitignore).

4

Abrir el firewall

Los puertos 80 y 443 deben estar abiertos al exterior:

$ sudo ufw allow 80/tcp
$ sudo ufw allow 443/tcp
$ sudo ufw status

Si el proveedor tiene además un firewall de red en su panel, abre 80 y 443 ahí también.

5

Arrancar

$ docker compose up -d --build

La primera vez tarda unos minutos (construye la imagen con LibreOffice). Al terminar, Caddy pide el certificado solo y en ~1 minuto tendrás https://planes.tudominio.com.

6

Comprobar

$ docker compose ps            # 'app' y 'caddy' en "running"
$ docker compose logs -f app   # "Planes app en http://localhost:3000"
$ docker compose logs caddy    # emisión del certificado TLS

Abre https://planes.tudominio.com: sale la pantalla de login, entras con un usuario de APP_USERS, pegas un boletín de prueba y generas un plan → debe descargarse el .docx.

Operaciones habituales

Tareas del día a día una vez desplegado.

# Ver logs de la app
$ docker compose logs -f app

# Añadir / quitar usuarios: edita APP_USERS en .env y recarga (SIN rebuild)
$ docker compose up -d

# Actualizar tras cambios de código
$ docker compose up -d --build

# Parar todo
$ docker compose down

Histórico de planes

Cada plan generado queda guardado y se puede buscar por año y mes.

  • En la app, enlace "Histórico" (arriba a la derecha): filtra por año y mes y descarga el Word/PDF de cualquier plan anterior.
  • El histórico y los ficheros generados viven en la carpeta outputs/ del servidor (montada como volumen), así que se conservan entre redespliegues. No borres esa carpeta si quieres mantener el histórico; puedes copiarla para hacer copia de seguridad.

Si algo falla

Los tropiezos más probables y su causa.

SíntomaCausa probable / solución
El navegador no carga o Caddy no saca certificadoEl DNS aún no resuelve al servidor (paso 1) o 80/443 cerrados (paso 4). Revisa docker compose logs caddy.
Login correcto pero vuelve al login (bucle)Falta NODE_ENV=production en .env, o se entra por http:// en vez de https://. Entra siempre por HTTPS.
Al generar: error de la API / 401ANTHROPIC_API_KEY vacía o incorrecta en .env. Corrígela y docker compose up -d.
Se genera el .docx pero no el PDF de previewEn la imagen LibreOffice ya viene, así que no debería pasar. El .docx es la descarga principal.

Seguridad — en breve

  • La app nunca se expone directamente: solo Caddy (80/443). La app escucha en el puerto interno 3000 dentro de la red de Docker.
  • La ANTHROPIC_API_KEY vive solo en .env en el servidor, jamás en el navegador.
  • El login tiene límite de intentos: 8 fallos en 15 min bloquean temporalmente esa IP.
  • Los .docx/.pdf generados se borran solos pasadas 48 h (configurable con RETENTION_HOURS en .env).
